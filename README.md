
## Описание программы

Программа предназначена для приближённого сравнения названий компаний (и ФИО) и ответа на вопрос: «могут ли эти две строки описывать одну и ту же организацию/персону».

- Сравнивает две строки с названием компании/банка/организации или ФИО и возвращает bool-значение: совпадают ли объекты логически.
- Учитывает несложные аббревиатуры, порядок слов, стоп-слова (ООО, ПАО, Банк и т.п.), номера филиалов и шум в виде спецсимволов и регистра.   

## Структура проекта

- **Main** – прикладная логика:
  - `CompanyNameComparer` – основной класс, реализующий сравнение двух строк.   
  - `Program` – консольный запуск тестов: загружает `test_cases.json`, прогоняет все кейсы и выводит результат по каждому и итоговую статистику.   
- **Algorithms** – отдельные алгоритмы:
  - `CompanyNameNormalizer` – нормализация строк (регистр, спецсимволы, стоп-слова).
  - `AbbreviationMatcher` – проверка, является ли одна строка аббревиатурой другой по заглвным буквам (ГКБ ↔ Городская Клиническая Больница и т.д.).
  - `NumbersMatcher` – сравнение числовых частей (номера филиалов, отделений, «№123» и т.п.).
  - `SortingMatcher` – сравнение с учётом перестановки слов (Альфа-Банк Воронеж ↔ Воронеж Альфа-Банк).
  - `LevenshteinSimilarity` – оценка похожести по расстоянию Левенштейна (опечатки, близкие написания).
- **Dictionaries**:
  - `StopWords` – список служебных слов (формы собственности, типы организации, общие слова вроде «банк», «филиал», «администрация»), которые выкидываются при нормализации.   
- `test_cases.json` – набор позитивных/негативных тестов, покрывающих банки, небанковские организации, физические лица, аббревиатуры, числа, сортировку слов и баг-кейсы.   

## Использование в своем коде

Чтобы внедрить программу, необходимо реализовать следующее:

- Подключить проект/сборку `CompanyMatcher` в свой .NET‑проект.
- Создать экземпляр `CompanyNameComparer` (или использовать статический метод, если он реализован).   
- Вызвать метод сравнения, передав две строки:  

```csharp
using CompanyMatcher.Main;

var comparer = new CompanyNameComparer();

bool areSame = comparer.AreSameCompany(
    "ПАО СБЕРБАНК",
    "Московский филиал ПАО Сбербанк"
);

// areSame == true
```

- Использовать результат как:
  - фильтр при дедупликации контрагента в базе;  
  - часть пайплайна очистки и склейки справочников;  
  - проверку совпадения при ручном вводе/поиске.   

## Тестирование и отладка программы
Для отладки и развития алгоритмов удобно запускать консольное приложение (`Program`) – оно прогоняет `test_cases.json` и показывает, какие кейсы прошли/упали, что помогает безопасно менять эвристики и словари.

- Проваленные тесты подсвечиваются красным цветом (ожидаемый результат отличается от фактического. Тест не пройден);
- Успешные тесты подсвечиваются зеленым цветом (ожидаемый результат совпадает с фактическим. Тест пройден);
- В конце отображается общая статистика по пройденным тестам.
